# TaskSense — 航空设备维护智能看板系统

## 项目目标

构建一个融合 **RAG 检索增强生成** 与 **AI Agent 自动化** 的航空设备维护看板系统。以航空/飞机维护为核心领域，利用其高度标准化的文档体系（ATA 100），打造"简洁但强大"的智能维护管理工具。

### 创新点

1. **RAG 知识库集成** — 将航空维修手册、法规、排故指南向量化，在看板内直接检索并引用
2. **Agent 自动化看板** — AI 自动分类优先级、预填工卡字段、建议操作步骤、生成报告
3. **人机协作模式** — AI 建议以幽灵文本呈现，Tab 接受 / Esc 拒绝，人类始终掌握最终决策权

---

## 领域选择：航空/飞机维护

### 选择理由

航空维护领域在所有设备维护领域中 **RAG 适配度最高（96/100）**，远超汽车（84）、能源（78）等领域。

| 优势维度 | 说明 |
|---------|------|
| **预设标准化** | ATA 100 规范定义了 100 个标准系统章节，是即用型实体层级 |
| **免费公共域文档** | FAA 航空维修手册（~400MB PDF）、EASA 800+ 资源库、CAAC 规章制度，零成本获取 |
| **结构化故障代码** | 8 位 ATA 故障码（如 32-31-01 = 起落架轮子/刹车/轮胎故障）形成天然知识图谱 |
| **程序多样性** | 每种设备涵盖描述、排故、拆装、测试、检查等多种程序类型 |
| **双语资源** | 中英文资料均丰富，ATA 章号在华语航空界通用 |

### ATA 100 章节体系

航空维修文档按 **5 大类 100 章** 标准组织：

| 类别 | 章节范围 | 示例 |
|------|---------|------|
| 飞机通用 | 00-19 | 05=时限/维修检查、12=勤务 |
| 机体系统 | 20-50 | 21=空调、24=电源、27=飞行控制、28=燃油、32=起落架 |
| 结构 | 51-57 | 52=舱门、53=机身、57=机翼 |
| 螺旋桨/旋翼 | 60-67 | 61=螺旋桨 |
| 动力装置 | 70-91 | 72=发动机、73=发动机燃油、79=滑油 |

**页面块类型**（每章内按功能分块，对应看板任务类型）：

| 页码范围 | 任务类型 | 看板映射 |
|---------|---------|---------|
| 001-099 | 描述与操作 | 参考资料 |
| 101-199 | 故障隔离 | 排故 |
| 201-299 | 维护实践 | 执行中 |
| 301-399 | 勤务 | 计划勤务 |
| 401-499 | 拆卸/安装 | 部件更换 |
| 501-599 | 调整/测试 | 测试 |
| 601-699 | 检查 | 检查 |

## 核心功能

### 一、看板管理

#### 看板列（9 列工作流）

```
Backlog → Triage → Scheduled → Ready → In Progress → Inspection → Parts Hold → Completed → Archived
```

| 列 | 含义 | WIP 限制 | AI 增强 |
|------|------|-----|--------|
| **Backlog** | 所有待处理任务 | ∞ | Agent 自动从排班/AD/SB 拉取 |
| **Triage** | AI 辅助优先级分类 | 10 | 🟢 AI 自动分类 AOG/A/B/C/D |
| **Scheduled** | 已排程到班次/日期 | ∞ | 🟢 AI 排程建议 |
| **Ready** | 资源确认、零件到位 | 20 | 🟢 零件自动锁定 |
| **In Progress** | 技术人员执行中 | 按班次容量 | 🟢 AI 操作步骤建议 |
| **Inspection** | RII/独立检查 | 15 | 🟡 合规检查提醒 |
| **Parts Hold** | 等待零件 | 10 | 🟢 到货自动提醒 |
| **Completed** | 已签批、待放行 | ∞ | 🟢 自动生成报告 |
| **Archived** | 已关闭归档 | ∞ | 数据分析/趋势 |

#### 卡片信息层级

每张看板卡片包含：

1. **优先级色条** — 左侧彩色条（红=AOG、橙=Cat A、黄=Cat B、蓝=Cat C、灰=Cat D）
2. **ATA 章节标签** — 可点击跳转到关联手册（RAG 检索入口）
3. **任务标题** — 故障描述或任务摘要
4. **飞机信息** — 注册号（尾号）· 机型 · 区域
5. **指派信息** — 技师 · 预估工时 · 截止时间
6. **任务类型徽章** — 排故/检查/勤务/拆卸安装/测试
7. **状态图标** — 评论数 · 附件数 · AI 警告标记

#### 三级渐进展开

| 级别 | 触发方式 | 显示内容 |
|------|---------|---------|
| L1 卡片 | 默认显示 | 标题、飞机、ATA、优先级、负责人、工时 |
| L2 悬浮预览 | 悬停 300ms | L1 + 摘要、子任务数、最近评论、关联文档 |
| L3 侧面板 | 点击卡片 | 完整详情、检查清单、AI 建议、RAG 检索、操作历史 |

#### 看板视图

- **看板视图**（主视图）— 列+卡片拖拽
- **列表视图** — 表格式紧凑显示
- **时间线视图** — 按日期/班次排列
- **日历视图** — 维护到期日可视化
- **机队总览** — 按飞机注册号分组的状态面板

#### Swimlane 分组维度

- 按 **ATA 章** 分组（系统维度）
- 按 **飞机注册号** 分组（资产维度）
- 按 **班组** 分组（人员维度）
- 按 **优先级** 分组（紧急维度）

### 二、RAG 知识库检索

#### 知识库内容

| 文档类型 | 来源 | 用途 |
|---------|------|------|
| 飞机维护手册（AMM） | FAA/EASA/CAAC 公共域 | 标准维护程序 |
| 故障隔离手册（FIM） | OEM 文档 | 排故决策树 |
| 咨询通告 AC 43.13-1B/2B | FAA | 检查与修理方法 |
| 适航指令（AD） | FAA/EASA/CAAC | 强制合规检查 |
| 服务通告（SB） | OEM（波音/空客等） | 建议性维护 |
| 最低设备清单（MEL） | 各航司 | 放行标准 |
| 历史工单记录 | 系统内积累 | 相似故障匹配 |

#### 检索模式

- **混合检索** — BM25 关键词匹配（精确搜零件号/故障码）+ 向量语义搜索（模糊搜症状描述）
- **层级感知** — 保留 ATA 章节层级上下文，返回整棵子树而非孤立段落
- **多模态支持** — 文本 + 表格 + 示意图统一检索

#### RAG 在看板中的嵌入点

| 入口 | 触发 | 效果 |
|------|------|------|
| 点击 ATA 标签 | 单击卡片上 ATA 章节号 | 直接检索该章节手册 |
| 右键菜单 | 右键 → "AI 查找相关文档" | 基于卡片内容检索 |
| Cmd+K 命令面板 | 自然语言输入 | 全局搜索 |
| 侧面板常驻区 | 点击卡片打开详情 | 自动展示相关知识片段 |
| 创建卡片时 | 输入故障描述 | 自动匹配 ATA 章 + 历史相似工单 |

### 三、AI Agent 自动化

#### Agent 角色体系

| Agent | 触发条件 | 功能 | 自主权级别 |
|-------|---------|------|-----------|
| **Triage Agent** | 新卡片进入 Backlog | 分析故障码/症状 → 自动分类优先级 + ATA 章节 | 低（全自动） |
| **Scheduling Agent** | 卡片进入 Scheduled | 建议最佳排程时间、检查资源冲突 | 中（建议+确认） |
| **Parts Agent** | 卡片创建时 | 预判所需零件 → 检查库存 → 标记短缺 | 低（全自动） |
| **Compliance Agent** | 涉及安全关键系统 | 检查 AD/SB 适用性、标记强制要求 | 高（草稿+审批） |
| **Suggest Agent** | 卡片处于执行列 | 推荐操作步骤、标准工卡模板 | 中（建议+确认） |
| **Report Agent** | 每日/每班次自动 | 生成完成统计、异常总结、趋势分析 | 低（全自动） |
| **Handover Agent** | 班次交接时 | 生成交接清单（进行中/阻塞/待跟进） | 中（草稿+确认） |
| **Anomaly Agent** | 持续监控 | 检测重复故障模式（同飞机同ATA重复） | 低（全自动告警） |

#### AI 自主权三级治理

航空维护受严格法规约束，AI 行为按风险分级控制：

| 级别 | 适用场景 | AI 行为 | 人类角色 |
|------|---------|--------|---------|
| **低自主权** | 非安全项（分类、排程建议、数据填充） | 全自动执行 | 可事后审查 |
| **中自主权** | 操作项（工单填写、操作建议） | 提议并等待确认 | 确认后生效 |
| **高自主权** | 安全/法规项（AD 合规判断） | 仅生成草稿，锁定待审批 | 必须审批 |

**硬约束**：AI 只能检索和引用已认证的手册内容，绝不允许从零生成维护程序。

#### AI 交互模式

- **内联幽灵文本** — 字段编辑时，400ms 防抖后以半透明显示 AI 建议，Tab 接受/Esc 拒绝
- **侧面板建议区** — 打开卡片详情时，独立区域展示 AI 操作建议、警告、预测
- **命令面板** — Cmd+K 唤起，支持自然语言查询和操作
- **主动告警** — 异常检测结果以通知形式推送到卡片/顶栏

### 四、告警与通知

#### 告警严重度分级

| 级别 | 样式 | 场景 | 关闭方式 |
|------|------|------|---------|
| **Critical** | 红色锁定弹窗 + 声音 | AOG 任务、AD 紧急合规、零件短缺导致停飞 | 必须确认 |
| **Warning** | 橙色顶部横幅 | 任务超时、WIP 超限、即将到期 MEL 项 | 5 秒自动消失 |
| **Info** | 蓝色轻提示 | Agent 分类完成、报告生成 | 3 秒自动消失 |
| **Inline** | 卡片内图标+徽章 | 零件待确认、合规警告、重复故障标记 | 持续显示 |

#### 通知渠道

- 应用内顶部横幅（Toast）
- 卡片内联徽章
- 通知中心面板（可回顾）
- 每日/班次摘要（邮件/应用内）

### 五、搜索与命令面板

**Cmd+K** 唤起命令面板，支持：

- **自然语言搜索** — "B-5823 起落架常见故障？"
- **导航跳转** — "跳转到 ATA 32 看板"
- **操作执行** — "生成今日班次报告"
- **RAG 查询** — "查看 737-800 起落架排故程序"

底层使用混合检索（BM25 + 向量 + 知识图谱）确保精确匹配。

## UI 设计原则

| # | 原则 | 说明 |
|---|------|------|
| 1 | **渐进暴露** | 默认简洁，详情按需展开（Linear 式 Space-for-Peek） |
| 2 | **键盘优先** | 所有操作可键盘完成，Cmd+K 为核心入口 |
| 3 | **上下文保持** | 三栏布局（导航→看板→详情），不丢失位置 |
| 4 | **智能预填** | Agent 自动填充 ATA 章、故障码、优先级 |
| 5 | **人机协作** | AI 建议幽灵文本呈现，Tab 接受到 Esc 拒绝 |
| 6 | **航空语义** | 颜色/图标/术语遵循航空维护行业习惯 |
| 7 | **离线可用** | 机库网络不稳定，PWA 离线+自动同步 |
| 8 | **暗色主题** | 专业暗色，减少机库/驾驶舱眩光 |

## 技术栈

| 层 | 方案 | 说明 |
|----|------|------|
| 前端/UI | Flet (Python) | 基于 Flutter，Material Design，暗色主题原生支持 |
| AI/RAG 框架 | LangChain + LangGraph | Agent 编排、RAG 链、工具调用 |
| 向量数据库 | Chroma / Qdrant | 嵌入存储与语义检索 |
| 嵌入模型 | 待定（支持中英文） | 文档向量化 |
| LLM | 待定（支持中英文） | Agent 推理、建议生成 |
| 数据存储 | SQLite（原型）/ PostgreSQL（生产） | 任务、飞机、用户数据 |
