# tools 综述
根据原型文档描述，细化出7个Agent功能，这8个功能可以在全局搜索内命令调用，也可以
在标题栏折叠工具栏下拉菜单内调用：

## 1、生成大纲
Agent根据系统提示词会或skill + 用户输入信息生成可生成具体任务对象清单的
详细任务大纲和描述，经可能多的包含详细信息以满足所有任务属性要求。Agent可
主动搜索和向用户索要信息。输出应该是格式化大纲文档文本，可通过单独弹窗
打开审核并保存到本地，同时在对应对话气泡内部提供点击入口。

## 2、生成任务
根据用户提供的大纲或上下文大纲，按要求生成一系列任务到待处理队列，AI添加的
任务以斜体幽灵文本任务存在，对应的对话气泡和任务队列头部、单独任务卡片会出现 接受|拒绝 选项，点击后消失，
用户可以一次全部同意或同意一部分或撤销同意。

## 3、自动分类
Agent 根据用户上下文、数据库、用户信息和大纲，自动为用户指定的或全部待处理任务添加优先级信息，
同样的类似生成任务，在已分类队列内幽灵文本显示，对应的对话气泡和任务队列头部、单独任务卡片会出现 接受|拒绝 选项，点击后消失，
用户可以一次全部同意或同意一部分或撤销同意。

## 4、自动排程
Agent 根据用户上下文、数据库、用户信息和大纲，自动为用户指定的或全部已分类任务添加优先级信息，
同样的类似生成任务，在已排程队列内幽灵文本显示，对应的对话气泡和任务队列头部、单独任务卡片会出现 接受|拒绝 选项，点击后消失，
用户可以一次全部同意或同意一部分或撤销同意。

## 5、自动验收
Agent 根据任务提交日志，给出每个待验收任务的建议和操作，审核操作类似上文的幽灵气泡+选项。

## 6、生成报表
Agent 根据任务看板状态、全部日志、提交日志等信息生成md格式报表并保存。

## 7、任务审核
Agent 读取大纲、上下文、知识库判断，给出已排程、已分类、待处理任务合规性判断和解读错误、问题，给出
单独审核结果入口和合适方式高亮对应任务卡片。

根据任务标题 ATA编号 任务描述 三者互相补全（给出其中一个就可以），类似IDE一样按tab按键接受补全（有提示）
