# UI 设计灵感参考

## 设计目标

> **"简洁但强大"** — 默认看起来像 Trello 一样简单，深度功能按需展开，键盘驱动高效操作。

---

## 优秀产品设计模式

### 1. Linear — 三栏布局 + 键盘驱动

Linear 是"简洁但强大"的典范。

**借鉴要点：**
- **三栏布局**：左侧可折叠导航 → 中间主内容区 → 右侧详情面板（点击滑出）
- **Space-for-Peek**：按住空格键悬浮预览项目详情，松开即关闭
- **Cmd+K 命令面板**：全局入口，搜索+导航+操作融合
- **键盘快捷键全覆盖**：所有操作可键盘完成，鼠标可选
- **极简卡片**：默认仅显示标题+少量标识，信息通过悬浮/展开呈现

**对我们的启发：** 看板主体只显示核心信息（标题+ATA章+优先级色条+尾号），更多信息通过 Shift+悬停或点击展开。

### 2. Notion — 属性系统 + 多视图

Notion 的数据库系统提供了灵活的信息组织。

**借鉴要点：**
- **属性即字段**：每张卡片可定义多种属性（文本、选择、日期、人员、文件等）
- **同一数据多视图**：看板/列表/时间线/日历/画廊自由切换
- **筛选+排序+分组**：三维组合形成任意数据切片
- **属性显示控制**：可控制卡片上显示哪些属性

**对我们的启发：** 任务属性体系（ATA章、优先级、飞机、班组等）支持多视图切换，看板卡片可自由配置显示字段。

### 3. Height — AI 原生集成

Height 是最接近我们目标的 AI 增强看板。

**借鉴要点：**
- **AI 侧栏**：独立面板展示 AI 建议，不干扰主看板
- **自动分类**：AI 自动为新任务打标签、分类、分配优先级
- **智能预填**：创建任务时自动补全字段
- **内联 AI**：在任务详情中输入 `/` 触发 AI 操作
- **自主 Agent**：可设定规则让 AI 自动执行特定操作

**对我们的启发：** AI 建议放在侧面板独立区域，字段预填用幽灵文本模式，Cmd+K 触发 AI 操作。

### 4. Huly — 卡片关联图谱

Huly 提供了任务间关系的可视化。

**借鉴要点：**
- **阻塞/被阻塞关系**：卡片间可建立依赖关系
- **子任务展开**：在卡片内直接查看子任务列表
- **关联图谱可视化**：查看任务间的网络关系

**对我们的启发：** 排故任务可能有前置依赖（如必须先检查A才能拆卸B），用卡片连线表示。

### 5. Trello — 极简拖拽

Trello 证明最简单的交互模式最持久。

**借鉴要点：**
- **零学习成本**：任何人看到都能理解
- **拖拽即操作**：拖到不同列 = 改变状态
- **Butler 自动化**：规则引擎自动执行重复操作

**对我们的启发：** 核心看板交互保持 Trello 式的直观，复杂功能藏在 Cmd+K 和右键菜单中。

## AI 交互模式详解

### 幽灵文本（Ghost Text）

**来源：** Cursor、GitHub Copilot、Obsidian AI

```
用户在"标题"字段输入：
┌──────────────────────────────────────┐
│ 起落架异响排查                        │ ← 用户输入
│ 前起落架转向异响，可能为转向          │ ← AI 建议（灰色/半透明）
│ 作动筒故障                            │
└──────────────────────────────────────┘

Tab → 接受建议    Esc → 拒绝   继续输入 → 自动消失
```

**实现要点：**
- 输入停止后 400ms 触发推理
- 建议文本 opacity: 0.4，与用户输入明确区分
- 使用轻量模型控制延迟在 150ms 以内
- 已接受的建议与手动输入不可区分

## 看板交互模式

### 拖拽行为

| 操作 | 效果 |
|------|------|
| 拖拽到另一列 | 改变任务状态 |
| 拖拽在列内排序 | 调整优先级顺序 |
| Shift+拖拽 | 复制卡片到目标列 |

## 航空特有 UI 惯例

### 信息层级

航空维护软件中信息优先级：

1. **尾号/注册号** — 始终作为主标识符，最大字体加粗
2. **飞机状态** — 运行中/维修中/AOG/延期故障
3. **截止时间** — 飞行小时/循环/日历三种到期方式
4. **工单类型** — 计划/非计划/AD/SB
5. **当前状态** — 计划/进行中/已完成/挂起

### 颜色惯例

航空业通用"交通灯"系统：

- **红** = AOG/停飞/逾期 — 不解决不能飞
- **琥珀/黄** = 注意/延期（MEL）/即将到期 — 有限制运行
- **绿** = 适航/就绪/有效 — 完全符合

### 终端/移动端适配

机库/停机坪环境对 UI 的特殊要求：

- **手套友好触摸**：最小触摸目标 44-48px
- **暗色模式**：低光环境必备
- **高对比度**：最小 12pt 字号，强对比标签
- **离线优先**：完整工单包无网络可使用
- **集成采集**：相机拍照、扫码、电子签名嵌入流程
- **语音输入**：口头报告故障，语音转文本

## 动画与微交互

| 场景 | 动画 | 目的 |
|------|------|------|
| 卡片拖拽 | 拖拽中轻微放大+阴影 | 提供可拖拽反馈 |
| 卡片入列 | 滑入+弹跳 | 状态变化可见 |
| AI 建议出现 | 淡入（300ms） | 不突兀 |
| 侧面板打开 | 从右滑入（200ms） | 保持空间连续性 |
| 命令面板 | 缩放淡入（150ms） | 快速响应 |
| 通知 Toast | 顶部滑入+3-5秒自动消失 | 通知但不打断 |
| 卡片删除 | 缩小淡出 | 可撤销 |
| 优先级改变 | 左侧色条过渡动画 | 视觉反馈 |

## 响应式布局断点

| 断点 | 宽度 | 布局 |
|------|------|------|
| **桌面** | ≥1280px | 三栏：导航 + 看板 + 侧面板 |
| **平板横屏** | 1024-1279px | 两栏：看板 + 侧面板（导航折叠为图标） |
| **平板竖屏** | 768-1023px | 单栏：看板（侧面板覆盖模式） |
| **手机** | <768px | 单栏：按列滑动，卡片列表模式 |
