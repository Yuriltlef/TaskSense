# 标题栏：
### 1. 区块重组与视觉去留（移除第二行）

*   **舍弃无用视觉信息**：彻底移除原界面的**文字标题**（“TaskSense — 航空维修智能看板”）以及**第二行**所有的状态数据（机队、运行中、维修中等），让顶部栏只作为纯粹的操作控制区。

### 2. 顶部栏布局重构（单行贯穿）

重构后的顶部栏建议采用**“左中右三段式”**视觉布局（参考 VS Code 顶栏的紧凑感）。

*   **左侧（工具与按钮组）**：
    保留“新建任务”按钮。将原第二行的“刷新”、“过滤”等图标按钮，直接平移放置到“新建任务”按钮的右侧，两者之间保持适当间距（如 16px）。左侧整体呈现为一个紧凑的轻量级工具托盘。
*   **中间/中偏右（核心搜索区）**：
    将原第二行的**搜索框**提上来，放在顶部栏的中间偏右位置，居中放置或靠右排列（参考 VS Code 中搜索框的位置）。为了视觉平衡，搜索框不要做得太大，采用竖向扁平的胶囊状或矩形微圆角，视觉长度控制在能容纳 10-15 个中文字符即可。
*   **右侧（系统设置区）**：
    保留最右侧的“设置”图标及用户/账号入口图标，它们在视觉上作为收尾。

### 3. 视觉细节参考（VS Code 风格的极简调性）

*   **色彩背景**：顶栏背景建议采用纯黑色或**极深的磨砂灰黑**（参考 VS Code 的顶栏底色，与下方工作区形成微弱的明暗对比）。
*   **分割线**：底部不要加粗大的分割线，仅保留一条 1px 宽、极高透明度（如 10% 透明度）的极细灰色分割线，将顶栏与下方工作区隔开，达到极简的效果。
*   **输入框样式**：搜索框内部保持深色底色，无外发光的厚重边框，仅保留 1px 极细的暗灰边框；搜索框内文字建议使用浅灰白色，占位符（Placeholder）文字使用较暗的灰色。
*   **图标与按钮**：“新建任务”按钮保留视觉强调色（蓝色），其它工具图标进行**扁平化处理**，统一使用白色/浅灰色的细线条风格，去除不必要的杂色和阴影。
*   **对齐与留白**：左侧的工具组和右侧的设置组紧贴左右边缘；搜索框两端留白充足，不要被挤压。

### 4. 交互视觉优化（非代码实现）

*   **悬浮反馈**：鼠标悬浮到工具图标（如刷新、设置）上时，视觉上呈现一个正方形的**细微浅灰底块**（建议参考 VS Code 的悬浮色），而不是改变图标颜色，这样能保持界面高度整洁。
*   **聚焦反馈**：当鼠标点击搜索框时，输入框的边框颜色可以产生视觉高亮（如微弱发蓝的细边框），以提示当前处于输入状态。

用这套视觉方案去调整后，您的顶部就会变成一行干净、利落、带有专业 IDE 气质的极简菜单栏。您可以把这些视觉描述传达给 UI 设计师，或者直接用设计工具规划出来。
### 1. 核心优化一：精简文字信息（缩短视觉长度）
这是解决文字挤占横向空间最有效的 UI 策略。在不失严谨的情况下，将冗余词组精简为**两字词**：
*   `机队` → 保留（2字）
*   `运行中` → 改为 `运行`（2字）
*   `维修中` → 改为 `维修`（2字）
*   `AOG` → 保留（外文术语，视觉短）
*   `逾期任务` → 改为 `逾期`（2字）
*   `未关闭故障` → 改为 `故障未关` 或 `未关故障`（压缩至4字，视觉比5个字紧凑很多）
*   *目的*：将所有指标的文字统一控制在 2~4 个字以内，节省大约 30% 的横向宽度，为多指标留出余量。

### 2. 核心优化二：利用“留白”进行“左右两段式”分组
在横向空间有限的情况下，**切忌6个指标从左到右平铺无间隔**。请将整行状态栏在视觉上拆分为**左、右两组**：
*   **左侧组（常态运行）**：包含 `3 机队`、`1 运行`、`1 维修`
*   **右侧组（预警紧急）**：包含 `1 AOG`、`1 逾期`、`4 故障未关`
*   **视觉分隔**：在两组之间**留出一段明显的空白区域（留白）**，甚至可以加一个极浅的竖线（透明度极低，作为软分隔）。这种设计能将用户的阅读视线强制打断，避免一串文字看晕。

### 3. 核心优化三：构建“数字突出，文字静默”的色彩层级
这是 VS Code 状态栏保持清爽的核心秘密，**数字是焦点，文字是底色**。
*   **数字`3/1/1`等**：使用**高亮、有体量感**的视觉色彩（如深邃的蓝色或清晰的白色），字号比文字略大 2px（如14px），视觉上强调数量。
*   **文字`机队/运行/维修`等**：使用**统一的浅灰**（如 60% 的白，`#999999`），字号缩小至 12px，不加粗。
*   **异常警告色**：对于紧急状态，**只改变数字的颜色**。比如 `1 AOG` 的数字用亮橙色，`4 故障未关` 的数字用红色。文字依然保持浅灰色。
*   *效果*：用户只需要扫一眼，看到“亮眼的有色数字”就知道出问题了，同时旁边的灰色文字又提供了精确的语义说明。

### 4. 核心优化四：保留底部的“物理分割线”，强化秩序感
状态栏因为包含众多文字和数字，如果悬浮在下方页面之上，会显得不够整齐。
*   **视觉细节**：第二行状态栏的**底部不要留白**，而是直接做一条 **1px 的极细、极高透明度**（10%-20%）的浅色分割线，将状态栏与下方主工作区（待处理列表）彻底隔开。
*   **背景色**：整行状态栏使用比主顶栏更黑一点的纯色底（`#141414` 或 `#181818`），让状态栏作为一个独立扁平的“信息带”存在。
